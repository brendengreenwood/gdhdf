Proposal

cgl

A developer experience CLI for Cargill. One command to authenticate, install, configure, and diagnose.
The Problem

Nobody owns the seams between teams.

Auth, registries, CI, proxy configs, internal SDKs — each team owns their slice of infrastructure. The developer is left to stitch it all together manually, every time, from scratch.

The Problem

Getting set up is painful.

Registry tokens — dig through SharePoint for PAT setup docs that may or may not be current
Proxy configs — npm, git, pip, Docker, curl all need separate proxy settings
Auth boilerplate — every internal SDK requires its own auth wiring and env vars
VPN + certs — corporate CA certs need to be in the right trust stores for every tool
Tribal knowledge — the real docs are in Dave's head and a Teams thread from 2023
Repeat per project — none of this carries over. New repo, new pain.
The Problem

What does this actually cost?

Every hour spent fighting configuration is an hour not spent building. The cumulative drag across every developer, every project, every quarter is significant.

The Solution

Put a front door on it.

$ cgl install sprout

A single CLI that authenticates once, resolves packages against internal registries, configures proxy and cert settings, and encodes tribal knowledge into repeatable commands. Distributed via Jamf as a single Rust binary. No bootstrapping problem.

The Solution

Core commands

cgl setup
First-run environment bootstrap. Installs Homebrew, configures proxy, installs CA certs, sets up internal registry access. One command, fully configured machine.
cgl auth login
OAuth via Azure AD. Opens browser, completes auth flow, caches token in system keychain. Silent refresh on all subsequent commands.
cgl install [pkg]
Resolves against internal registry, handles auth transparently, pulls artifact, runs post-install hooks for config scaffolding.
cgl doctor
Diagnostic check. Auth status, proxy config, registry access, Node/Python versions, VPN status. Replaces pinging Dave.
cgl scaffold [tmpl]
Project templates with all internal plumbing pre-wired — CI config, auth boilerplate, linting, Docker, .npmrc.
cgl update
Self-update and dependency refresh. When IT changes proxy settings, push a cgl update through Jamf — one command fixes everyone.

  5. First commit — day 1
